# Base65536 文件编码解码工具

## 概述

Base65536 是一种使用 Unicode 字符（0x10000-0x1FFFF）的编码方案，可以将任意二进制数据编码为可读的文本字符串。此工具支持 gzip 压缩和原文件名保存，适用于文件传输和数据备份场景。

## 快速开始

### 安装依赖

```bash
pip install base65536
```

### 基本用法

```bash
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
Base65536 文件编码/解码工具（带置换加密和元数据隐藏）
使用Base65536编码将任意文件转换为Unicode文本，支持gzip压缩、原文件名保存，
以及基于SHA-256密钥的字符级置换加密，加密模式下隐藏真实元数据。

安装依赖:
    pip install base65536

用法:
    # 普通编码（无加密，元数据明文）
    python skill.py encode <输入文件> [-o <输出文件>] [--no-compress]

    # 加密编码（打乱字符顺序，隐藏元数据，输出密钥）
    python skill.py encode <输入文件> --scramble [-o <输出文件>]

    # 解码（若文件已加密，需提供密钥）
    python skill.py decode <输入文件> --key <密钥整数> [-o <输出文件>]

示例:
    # 加密编码
    python skill.py encode secret.zip --scramble -o public.txt
    # 输出:
    #   🔑 密钥（整数）: 108544482569932551567348223456789012...
    #   ✓ 已编码: public.txt

    # 解密还原
    python skill.py decode public.txt --key 108544482569932551567348223456789012...

注意:
    - 加密模式下，文件内的元数据（文件名、大小等）被隐藏，仅显示假占位信息。
    - 请妥善保管密钥，丢失密钥无法恢复文件。

Author: Assistant
Version: 3.0.0
"""

```

## 工作流程

### 编码流程

1. 读取原始二进制文件
2. 可选：使用 gzip 压缩数据（默认启用，压缩级别 9）
3. 使用 Base65536 编码为 Unicode 文本
4. 添加 `#METADATA:` 头部保存原文件名和压缩状态
5. 保存为 `.b65536.txt` 文件

### 解码流程

1. 读取编码文本文件
2. 解析 `#METADATA:` 头部获取元信息
3. 使用 Base65536 解码为二进制数据
4. 自动检测并解压 gzip 压缩数据
5. 使用元数据中的原文件名保存文件

## 文件格式

编码后的文本文件格式：

```
#METADATA:{"original_name": "原始文件名", "compressed": true, "original_size": 12345}
[Base65536 编码数据...]
```

### 编码 PDF 文件

## 适用场景

- **跨平台文件传输**：编码后的文本可以在任何支持文本的平台传输（邮件、聊天、代码注释）
- **隐写术**：结合图片分享等方式传输文件
- **数据备份**：将二进制数据转换为文本进行备份
- **API 传输**：在纯文本 API 中传输二进制数据
- **代码嵌入**：将文件内容嵌入代码或配置文件

## 技术规格

- **编码方案**：Base65536（使用 Unicode 字符 0x10000-0x1FFFF）
- **压缩算法**：gzip（zlib，压缩级别 9）
- **元数据格式**：JSON，附加在文件首行
- **自动检测**：解码时自动检测 gzip 压缩，无需手动指定

## 性能特性

| 原始类型 | 压缩效果 | Base65536 膨胀率 | 综合效果 |
|---------|---------|-----------------|---------|
| 文本文件 | 70-80% | ~50% | 35-40% |
| 图片/视频 | 90-99% | ~50% | 45-99% |
| 已压缩文件 | 无 | ~50% | ~50% |

## 注意事项

1. Base65536 编码后约为原始数据的 50%（因为每个字符约 2 字节）
2. 启用 gzip 压缩可以显著减少数据量，特别是对于文本文件
3. 对于已压缩的文件（如 .zip、.jpg），不建议再启用压缩
4. 解码时自动检测压缩格式，无需手动指定
